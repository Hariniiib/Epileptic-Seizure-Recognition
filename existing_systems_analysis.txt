---

## 📌 Slide 1: Introduction — The Problem

Epilepsy affects over 50 million people worldwide (WHO), making it one of the most common neurological disorders globally.

- Seizures are caused by abnormal, excessive electrical activity in the brain.
- Electroencephalography (EEG) is the primary diagnostic tool — it records the brain's electrical signals through electrodes placed on the scalp.
- Early and accurate detection of seizures is critical for timely treatment and preventing brain damage.

### Why Automation is Needed:
| Challenge | Impact |
|:---|:---|
| Manual EEG review is extremely time-consuming | A single 24-hour recording takes hours to review |
| Subjective — different neurologists may give different interpretations | Inconsistent diagnoses |
| Shortage of neurologists globally, especially in developing countries | Delayed diagnosis and treatment |
| Routine EEG lasts only 20–60 minutes — seizures may not occur | Missed seizures, false negatives |
| Real-time monitoring is not feasible with manual inspection | No immediate alerts for emergencies |

## 📌 Slide 3: System 1 — Manual Visual EEG Inspection (Traditional Gold Standard)

### How It Works:
- A trained neurologist or electroencephalographer visually inspects raw EEG waveforms on a screen.
- They look for characteristic patterns: sharp waves, spikes, spike-and-wave complexes, and other epileptiform discharges.
- Diagnosis is made based on pattern recognition, clinical context, and professional experience.

### Efficiency:
| Metric | Value |
|:---|:---|
| Accuracy | Depends on expertise — estimated 70–85% for general neurologists |
| Speed | 1–4 hours to review a 24-hour EEG recording |
| False Positive Rate | High — artifacts (eye blinks, muscle movement) mimic seizure patterns |
| Real-Time Capability | ❌ Not feasible for continuous monitoring |
| Scalability | ❌ Cannot scale — requires one expert per patient |

### Key Limitations:
1. Subjectivity — High inter-observer variability (different doctors, different conclusions)
2. Human Error — Fatigue, distraction lead to missed seizures
3. Time-Intensive — Impractical for ICU or long-term monitoring
4. Over-reading — Inexperienced readers may misinterpret normal EEG variants as pathological
5. Resource Bottleneck — Critical shortage of qualified EEG experts worldwide

## 📌 Slide 4: System 2 — Classical Machine Learning Methods

These methods require manual feature extraction by domain experts before classification.

### Common Pipeline:
```
Raw EEG Signal → Feature Extraction (DWT, FFT, PCA, Entropy) → Classifier (SVM/KNN/RF) → Prediction
```

### 2.1 Support Vector Machine (SVM)
| Aspect | Details |
|:---|:---|
| How It Works | Finds the optimal hyperplane that maximally separates seizure vs. non-seizure classes in feature space |
| Feature Extraction | Requires manual features — Discrete Wavelet Transform (DWT), statistical features, entropy measures |
| Accuracy (Binary) | 90% – 99% (depends heavily on kernel choice and feature engineering) |
| Accuracy (5-Class) | 80% – 93% |
| Strengths | Works well on small datasets; mathematically well-founded |
| Weaknesses | Poor scalability; sensitive to kernel/parameter choices; requires expert-crafted features |

### 2.2 K-Nearest Neighbors (KNN)
| Aspect | Details |
|:---|:---|
| How It Works | Classifies a new sample based on the majority class among its K nearest training neighbors |
| Accuracy (Binary) | 90% – 96% |
| Accuracy (5-Class) | 82% – 90% |
| Strengths | Simple to implement; no training phase; intuitive |
| Weaknesses | Slow inference (compares against all training samples); sensitive to irrelevant features; poor with high-dimensional data |

### 2.3 Random Forest (RF)
| Aspect | Details |
|:---|:---|
| How It Works | Ensemble of decision trees; each tree votes, and the majority wins |
| Accuracy (Binary) | 95% – 98.5% |
| Accuracy (5-Class) | 85% – 95% |
| Strengths | Robust to overfitting; handles high-dimensional data; provides feature importance |
| Weaknesses | Still requires manual feature extraction; limited ability to learn complex temporal patterns |

### 2.4 Decision Tree
| Aspect | Details |
|:---|:---|
| How It Works | Builds a tree of if-else rules based on feature thresholds |
| Accuracy (Binary) | 85% – 94% |
| Accuracy (5-Class) | 75% – 88% |
| Strengths | Highly interpretable; fast inference |
| Weaknesses | Prone to overfitting; unstable (small data changes can alter the tree significantly) |

### Summary: Limitations of All Classical ML Methods
> [!WARNING]
> Critical Drawback: All classical ML methods depend on manual, handcrafted feature extraction. This requires deep domain expertise in signal processing (DWT, FFT, entropy), and the quality of detection is only as good as the features chosen. They cannot automatically discover hidden patterns in raw EEG data.

## 📌 Slide 5: System 3 — Deep Learning Approaches

Deep learning eliminates the need for manual feature extraction — models learn directly from raw data (end-to-end learning).

### 3.1 1D Convolutional Neural Network (1D-CNN)
| Aspect | Details |
|:---|:---|
| How It Works | Slides small learnable filters across the 1D EEG signal to detect local spatial patterns (spikes, dips, sharp waves) |
| Accuracy (Binary) | 98% – 99% |
| Accuracy (5-Class) | 88% – 95% |
| Strengths | Automatic feature extraction; fast training; efficient for time-series data |
| Weaknesses | Cannot capture long-range temporal dependencies (a spike at time t=10 and its effect at t=150) |

### 3.2 LSTM / BiLSTM (Long Short-Term Memory)
| Aspect | Details |
|:---|:---|
| How It Works | Recurrent network that processes the EEG signal sequentially, maintaining a memory of past values. BiLSTM reads both forward and backward. |
| Accuracy (Binary) | 97% – 99% |
| Accuracy (5-Class) | 85% – 93% |
| Strengths | Excellent at capturing temporal dependencies; BiLSTM uses both past and future context |
| Weaknesses | Slow training (sequential processing); cannot extract spatial features independently |

### 3.3 VGG16 (Transfer Learning)
| Aspect | Details |
|:---|:---|
| How It Works | Pre-trained on ImageNet (1.2M images). EEG signals are converted to 2D spectrograms, then VGG16 extracts spatial features. |
| Architecture | 16 layers deep; 138 million parameters |
| Accuracy (Binary) | 96% – 99% |
| Accuracy (5-Class) | 84% – 92% |
| Strengths | Leverages powerful pre-trained features (transfer learning); recognizes textures and patterns in spectrograms |
| Weaknesses | Very large model (high compute); requires EEG→2D conversion; prone to overfitting on small EEG datasets |

### 3.4 ResNet50 (Transfer Learning)
| Aspect | Details |
|:---|:---|
| How It Works | 50-layer CNN with skip/residual connections that prevent vanishing gradients. Also uses 2D spectrograms. |
| Architecture | 50 layers deep; 25.6 million parameters; residual learning |
| Accuracy (Binary) | 97% – 99% |
| Accuracy (5-Class) | 85% – 93% |
| Strengths | Deeper than VGG16 without gradient problems; learns hierarchical features efficiently |
| Weaknesses | Still requires 2D conversion; computationally heavy; risk of overfitting on EEG |

### 3.5 Hybrid Models (CNN + LSTM/BiLSTM)
| Aspect | Details |
|:---|:---|
| How It Works | CNN branch extracts spatial features → LSTM/BiLSTM branch captures temporal dependencies → Features are fused for final classification |
| Accuracy (Binary) | 99% – 99.8% |
| Accuracy (5-Class) | 88% – 96% |
| Strengths | Combines the best of both worlds — spatial + temporal; state-of-the-art results |
| Weaknesses | Complex architecture; higher training time; requires careful hyperparameter tuning |

## 📌 Slide 6: System 4 — Commercial Clinical Software

These are proprietary, FDA-cleared software tools used in hospitals alongside EEG monitoring equipment.

### 4.1 Persyst (by Persyst Development Corp.)
| Aspect | Details |
|:---|:---|
| Type | FDA-cleared clinical EEG analysis software |
| Technology | Deep learning-based seizure detection + spike detection |
| Key Feature | "Seizure Probability Trends" — provides continuous probability scores |
| Sensitivity | 80% – 95% (varies by seizure type and software version P13/P14/P15) |
| False Alarm Rate | 0.5 – 2.0 per hour (varies by setting) |
| Integration | Built into Natus EEG systems |
| Limitation | Decision-support only — NOT a diagnostic replacement; performance varies significantly across seizure types |

### 4.2 Encevis (by AIT Austrian Institute of Technology)
| Aspect | Details |
|:---|:---|
| Type | Independent clinical EEG analysis software |
| Technology | AI-driven seizure detection, spike detection, artifact reduction |
| Key Feature | NeuroTrend — comprehensive EEG trending and visualization |
| Sensitivity | Comparable to Persyst (~80% – 95%) |
| Limitation | Primarily a clinical aid; cannot replace neurologist review; requires expensive EEG hardware |

### 4.3 Natus autoSCORE
| Aspect | Details |
|:---|:---|
| Type | AI-based comprehensive EEG assessment tool |
| Technology | Classifies entire EEG studies as normal/abnormal |
| Key Feature | Sub-classifies abnormalities; provides study-level assessment |
| Limitation | Designed for routine EEG reports, not real-time seizure detection |

> [!IMPORTANT]
> All commercial systems are decision-support tools only. They assist neurologists but cannot replace clinical judgment. They also require expensive proprietary hardware (₹15–50 Lakhs), making them inaccessible for smaller hospitals and developing regions.

### 5.1 Empatica Embrace2

| Technology | Electrodermal Activity (EDA) + Accelerometer (NOT EEG-based) |

| Sensitivity | 92% – 100% for tonic-clonic seizures |
| False Alarm Rate | 0.2 – 1.35 false alarms per 24 hours |
| Alert System | Sends SMS/call to caregivers via paired smartphone |
| Prescription | Required — must be ordered by a doctor |
| Cost | ~$249 + monthly subscription |

### 5.2 EpiMonitor (Successor to Embrace2)
| Aspect | Details |
|:---|:---|
| Type | Next-generation wearable seizure monitor by Empatica |
| Technology | Enhanced multi-sensor fusion |
| Status | Transitioning from Embrace2 platform |

### Limitations of Wearable Devices:
- ❌ Cannot detect non-convulsive seizures (absence, focal seizures have no shaking)
- ❌ Not a diagnostic tool — monitoring only
- ❌ Cannot predict seizures — only detects during occurrence
- ❌ Requires Bluetooth + smartphone proximity
- ❌ Does not use EEG data — cannot classify seizure types
